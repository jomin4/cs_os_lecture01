Sprinlab 유튜버의 **운영체제(OS) 강의 실습 저장소**입니다. 실습 언어는 **Python**이며, 강의 챕터별로 최상위 폴더를 나눠 진행합니다.

- GitHub 원격: https://github.com/jomin4/cs_os_lecture01 (public)
- 폴더 명명: 강의 제목 그대로. 예) `Lecture06-Process Synchronization and Mutual Exclusion-3`

---

## 학습 진행 방식 (매 챕터 공통)

각 강의는 아래 순서로 진행한다. 이 흐름을 건너뛰지 말 것.

1. **사용자가 먼저 강의를 시청**한다.
2. 사용자가 **강의 자료(슬라이드/캡처)를 Claude에게 공유**한다.
3. Claude와 **함께 개념을 이해**한다 — 이때 아래 "시각화 원칙"에 따라 인터랙티브 위젯으로 설명한다.
4. **어떤 부분을 실습할지 함께 결정**한다.
5. Claude가 **실습할 Python 코드를 제공**한다.
6. 사용자가 **직접 타이핑하며 학습**한다. (Claude가 코드를 통째로 대신 작성해 넣지 않는다 — 사용자가 손으로 익히는 것이 목적)

> 개념 이해(3~4단계)를 건너뛰고 완성된 해답을 먼저 던지지 말 것.

---

## 시각화 원칙

- OS 개념을 설명할 땐 **인터랙티브 위젯**(`mcp__visualize__show_widget`, 먼저 `read_me` 호출)을 **기본으로** 사용한다. 정적 텍스트/다이어그램만으로 끝내지 않는다.
- 위젯은 **채팅창에 바로 렌더링**한다. (별도 HTML 대시보드 파일로 모아 관리하지 않는다 — 예전에 `dashboard/index.html`을 만들었으나 번거로워서 폐기함)
- 위젯은 **직접 조작하며 원리를 스스로 발견**하도록 만든다: 버튼/슬라이더/상태 표시. 실제 메커니즘을 JS 상태머신으로 모델링한다.
- 공유 상태 실시간 표시 + 행위자별 상태 + 동작 로그를 넣어 원인→결과가 보이게 한다.
- 설명 산문은 채팅 응답에 쓰고, 위젯 안에는 넣지 않는다.
- 참고: 사용자는 이 도구를 "winget"이라 부르지만 실제로는 **widget(시각화) 도구**를 뜻한다. Windows `winget` 패키지 매니저가 아니다.

## Git 자동 처리

- 강의 진행에 따라 `init` / `add` / `commit` / `push`를 **자동으로** 처리한다. 매번 확인받지 않는다. (사용자가 이 저장소의 강의 실습 git 워크플로에 대해 명시적으로 위임함)
- 커밋 후에는 무엇을 커밋했는지 한 줄로 알린다.
- 단, force push·히스토리 재작성 등 되돌리기 어려운 작업은 먼저 확인한다.
- 커밋 메시지는 어떤 강의/무엇을 했는지 알 수 있게 작성한다.
